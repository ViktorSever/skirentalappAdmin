# Описание проекта: сайт управления бронированиями (JS + Firebase + Cupertino UI)

Этот документ описывает, что делает проект, из каких частей он состоит и как устроена логика работы.

---

## 1. Назначение проекта

Проект — это простое веб‑приложение для **просмотра и управления заявками на бронирование** (например, прокат горнолыжного оборудования) в стиле **Cupertino / iOS**.

Основные задачи:

- Показывать оператору список всех заявок из **Firebase Firestore**.
- Позволять быстро фильтровать заявки по статусу (новые, активные, завершённые, отменённые).
- Открывать подробную карточку заявки с полным набором данных.
- Менять статус заявки (выдача, отмена, завершение) с обновлением в Firestore.

---

## 2. Технологии

- **JavaScript (чистый, без фреймворков)** — логика фронтенда.
- **HTML + CSS** — верстка и стилизация в стиле Cupertino (iOS‑подобный UI).
- **Firebase Firestore** — облачная база данных для хранения заявок.
- **Firebase Web SDK (модульный)** — доступ к Firestore из браузера.
- **http-server** (npm‑пакет) — простой HTTP‑сервер для локального запуска.

---

## 3. Модель данных (коллекция `booking` в Firestore)

Каждый документ в коллекции `booking` описывает одну заявку:

```javascript
{
  id: string,              // ID документа (в Firestore хранится в meta, а не как поле)
  equipmentType: string,   // Тип оборудования (например, "Лыжи")
  gender: string,          // Пол
  age: number,             // Возраст
  heightCm: number,        // Рост (см)
  weightKg: number,        // Вес (кг)
  shoeSize: number,        // Размер обуви
  hatSizeCm: number,       // Размер головного убора (см)
  fullName: string,        // ФИО
  phone: string,           // Телефон
  date: string,            // Дата
  timeSlot: string,        // Временной слот
  status: "NEW" | "CONFIRMED" | "COMPLETED" | "CANCELLED"
}
```

Ключевое поле — `status`:

- `NEW` — новая заявка.
- `CONFIRMED` — подтверждённая / активная, оборудование выдано.
- `COMPLETED` — заявка завершена, оборудование возвращено.
- `CANCELLED` — заявка отменена.

---

## 4. Структура файлов

В корне проекта:

- **`index.html`** — главная страница со списком заявок и табами.
- **`details.html`** — страница деталей одной заявки.
- **`style.css`** — общие стили в стиле Cupertino:
  - стеклянные карточки,
  - плавные тени,
  - сегмент‑контрол для табов,
  - цветные индикаторы статуса,
  - тост‑уведомления.
- **`firebase.js`** — инициализация Firebase и функции работы с Firestore:
  - `fetchAllBookings()` — получить все заявки,
  - `fetchBookingById(id)` — получить одну заявку по ID,
  - `updateBookingStatus(id, newStatus)` — обновить статус заявки.
- **`ui.js`** — слой интерфейса:
  - рендер списка заявок,
  - рендер страницы деталей,
  - создание индикатора статуса и подсказок,
  - показ тост‑уведомлений.
- **`app.js`** — бизнес‑логика:
  - определяет, какая страница загружена (список или детали),
  - инициализирует соответствующие обработчики,
  - связывает `firebase.js` и `ui.js`.
- **`README.md`** — подробное руководство по созданию и устройству проекта.
- **`RUN.md`** — краткая инструкция по запуску и проверке.
- **`PROJECT.md`** — (этот файл) описание проекта.
- **`package.json`** — настройки npm и скрипты (`npm run start`).

## 5. Поведение главной страницы (`index.html`)

### Интерфейс

- Верхняя шапка:
  - заголовок «Бронирования»,
  - подзаголовок «Управление заявками».
- Блок табов (фильтры по статусу):
  - **Все** (`ALL`),
  - **Новые** (`NEW`),
  - **Активные** (`CONFIRMED`),
  - **Завершённые** (`COMPLETED`),
  - **Отменённые** (`CANCELLED`).
- Стеклянная карточка со списком:
  - счётчик заявок,
  - список элементов,
  - пустое состояние, если заявок нет.

### Элемент списка заявки

Каждый элемент строится из объекта `booking`:

- Заголовок: **`fullName — phone`**.
- Подзаголовок: **`date • timeSlot`**.
- Справа — цветной индикатор статуса:
  - `NEW` — сине‑фиолетовый градиент,
  - `CONFIRMED` — жёлто‑оранжевый,
  - `COMPLETED` — зелёный,
  - `CANCELLED` — красный.
- При наведении курсора на индикатор появляется подсказка с русским названием статуса:
  - «Новый», «Подтверждён», «Завершён», «Отменён».
- При клике на элемент — переход на `details.html?id=<ID>` с деталями заявки.

### Логика

- При загрузке страницы:
  - `app.js` вызывает `fetchAllBookings()` из `firebase.js`.
  - Затем `renderBookingList(allBookings, currentFilter)` из `ui.js`.
- При клике по табам:
  - меняется активный фильтр,
  - список перерисовывается с учётом выбранного статуса.

## 6. Поведение страницы деталей (`details.html`)

### Интерфейс

- Шапка:
  - кнопка «Назад» (возвращает на `index.html`),
  - заголовок «Заявка»,
  - подзаголовок с датой и временным слотом.
- Верхний блок:
  - ФИО (`fullName`),
  - телефон (`phone`),
  - индикатор статуса с подсказкой.
- Список полей в формате «название — значение»:
  - Тип оборудования,
  - Пол,
  - Возраст,
  - Рост (см),
  - Вес (кг),
  - Размер обуви,
  - Размер головного убора (см),
  - Дата,
  - Временной слот,
  - Статус (в русской форме: «Новый», «Подтверждён» и т.д.).
- Блок кнопок действий (динамически).

### Динамические кнопки

В зависимости от `status` заявки:

- `NEW`:
  - **«Отмена»** → меняет статус на `CANCELLED`.
  - **«Выдача»** → меняет статус на `CONFIRMED`.
- `CONFIRMED`:
  - **«Забрать»** → меняет статус на `COMPLETED`.
- `CANCELLED` или `COMPLETED`:
  - Кнопки действий не отображаются.

### Логика

- При загрузке:
  - `app.js` читает `id` из query‑параметров (`?id=...`).
  - Вызывает `fetchBookingById(id)`:
    - Если заявка не найдена — показывает тост‑ошибку.
    - Если найдена — рендерит детали и кнопки.
- При нажатии кнопки изменения статуса:
  - Статус локально обновляется (оптимистично),
  - Вызывается `updateBookingStatus(id, newStatus)` в Firestore,
  - При успехе:
    - тост «Статус заявки успешно обновлён.»,
    - состояние интерфейса соответствует новому статусу.
  - При ошибке:
    - статус откатывается,
    - показывается тост‑ошибка.

## 7. Стиль интерфейса (Cupertino / iOS)

Весь UI стилизован под дух iOS:

- **Стеклянные карточки**:
  - полупрозрачный белый фон,
  - эффект размытия (backdrop-filter),
  - мягкие тени.
- **Круглые углы**:
  - большие радиусы у карточек и кнопок,
  - табы с закруглением до «пилюль».
- **Сегмент‑контрол для табов**:
  - фон‑капсула с лёгким градиентом,
  - активный таб подсвечен синим цветом (как на iOS).
- **Минималистичные элементы**:
  - без лишних бордеров и ярких линий,
  - приоритет на плавность и читабельность.
- **Шрифт**:
  - `Inter` + системные шрифты (`-apple-system`, `SF Pro Text`) — близко к San Francisco.
- **Анимации**:
  - плавные hover‑эффекты у кнопок и карточек,
  - мягкое появление тост‑уведомлений и подсказок.

## 8. Для кого этот проект

Проект рассчитан на:

- начинающих разработчиков с базовыми знаниями JavaScript и HTML/CSS,
- тех, кто хочет быстро сделать простой административный интерфейс к Firestore,
- любителей iOS‑стиля, которым нужен аккуратный, лёгкий и понятный UI.

---

## 9. Как использовать этот проект

- В качестве **реального инструмента**:
  - Можно использовать как лёгкую админку для небольшого пункта проката,
    где заявки создаются через другую систему/форму, а здесь только
    просматриваются и изменяется статус.
- В качестве **учебного примера**:
  - Понять, как фронтенд общается с Firestore.
  - Попробовать паттерн разделения на слои `firebase.js` (данные),
    `ui.js` (отрисовка) и `app.js` (бизнес‑логика).
  - Поэкспериментировать с Cupertino‑стилем без использования UI‑фреймворков.
