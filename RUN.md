# Инструкция по запуску и проверке сайта бронирований (JS + Firebase + Cupertino UI)

Этот файл — короткий чек‑лист, как запустить и протестировать сайт локально.

---

## 1. Предварительные условия

- Установлены **Node.js** и **npm** (проверьте в терминале):

```bash
node -v
npm -v
```

- В Firebase Console уже создан проект с включённым **Cloud Firestore**.
- В коллекции **`booking`** есть тестовые документы с полями из модели:
  `equipmentType`, `gender`, `age`, `heightCm`, `weightKg`, `shoeSize`,
  `hatSizeCm`, `fullName`, `phone`, `date`, `timeSlot`, `status`.
- В файле `firebase.js` уже подставлен ваш `firebaseConfig` (в этом проекте он задан).

Ожидаемый результат: всё выше выполнено, ошибок при проверке `node -v` и `npm -v` нет.

---

## 2. Установка зависимостей

Перейдите в папку проекта (у вас это `/Users/viktor/BookingApp`):

```bash
cd /Users/viktor/BookingApp
```

Установите зависимости (http-server уже указан в `devDependencies`):

```bash
npm install
```

Ожидаемый результат: появится папка `node_modules`, команда завершается без ошибок.

---

## 3. Запуск локального сервера

В `package.json` уже настроен скрипт:

```json
"scripts": {
  "start": "http-server -p 5173 -c-1 .",
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

Запустите сервер:

```bash
npm run start
```

Если всё успешно:
- В терминале появится информация от **http-server** о том, что он слушает порт `5173`.

---

## 4. Открытие сайта в браузере

### Главная страница (список заявок)

Откройте в браузере:

```text
http://localhost:5173/index.html
```

Вы должны увидеть:
- Заголовок «Бронирования», подзаголовок «Управление заявками».
- Табы: «Все», «Новые», «Активные», «Завершённые», «Отменённые».
- Стеклянную карточку со списком заявок (или пустое состояние, если документов нет).

Если в `booking` есть данные:
- В списке появятся элементы вида:  
  **ФИО — телефон** и строка вида `дата • временной слот`.
- Справа у каждой заявки — цветной индикатор статуса.

---

## 5. Проверка фильтрации по статусам

На `index.html`:

1. Нажмите на таб **«Новые»**:
   - Должны отображаться только заявки со статусом `NEW`.
2. Нажмите на таб **«Активные»**:
   - Видны заявки со статусом `CONFIRMED`.
3. Нажмите **«Завершённые»**:
   - Видны заявки со статусом `COMPLETED`.
4. Нажмите **«Отменённые»**:
   - Видны заявки со статусом `CANCELLED`.
5. Нажмите **«Все»**:
   - Видны все заявки.

Если для выбранного статуса документов нет:
- Появится блок «Пока нет заявок. Измените фильтр или добавьте данные в Firestore».

Ожидаемый результат: фильтрация работает, активный таб подсвечивается.

---

## 6. Проверка всплывающих подсказок статуса

На `index.html` наведите курсор на цветной индикатор статуса любой заявки:

- Появится тёмная «облачко‑подсказка» с текстом на русском:
  - `Новый` / `Подтверждён` / `Завершён` / `Отменён`.

Ожидаемый результат: подсказка появляется и исчезает плавно при уходе курсора.

---

## 7. Открытие страницы деталей заявки

1. На `index.html` кликните по любому элементу списка.
2. Откроется URL вида:

```text
http://localhost:5173/details.html?id=<ID_документа>
```

На странице `details.html` вы увидите:
- В шапке кнопку «Назад».
- Заголовок «Заявка» и подзаголовок с датой и временным слотом.
- В верхнем блоке — ФИО и телефон.
- Справа — цветной индикатор статуса с подсказкой.
- Ниже — список полей:
  - «Тип оборудования — …»
  - «Пол — …»
  - «Возраст — …»
  - и т.д., включая «Статус — Новый/Подтверждён/…».

Ожидаемый результат: данные полностью соответствуют документу в Firestore.

---

## 8. Проверка кнопок изменения статуса

На `details.html` для разных статусов:

- Если статус `NEW`:
  - Видны кнопки **«Отмена»** и **«Выдача»**.
- Если статус `CONFIRMED`:
  - Видна кнопка **«Забрать»**.
- Если статус `CANCELLED` или `COMPLETED`:
  - Кнопок нет.

### Изменение статуса и проверка в Firestore

1. Откройте документ со статусом `NEW`.
2. Нажмите:
   - **«Отмена»** → статус должен стать `CANCELLED`.
   - или **«Выдача»** → статус должен стать `CONFIRMED`.
3. Перейдите в Firebase Console → Firestore → коллекция `booking`:
   - Убедитесь, что поле `status` у этого документа обновилось.
4. Обновите страницу `details.html`:
   - Кнопки должны соответствовать новому статусу
     (например, для `CANCELLED` — исчезнуть).

Ожидаемый результат:
- Статус в Firestore и на странице совпадает.
- После изменения появляется тост «Статус заявки успешно обновлён.».

---

## 9. Проверка обработки ошибок

Для теста можно временно «сломать» конфигурацию:

1. Откройте `firebase.js` и временно измените, например, `projectId` на неверный.
2. Обновите страницу `index.html`.

Ожидаемое поведение:
- Появится тост‑ошибка:
  - «Не удалось загрузить список заявок. Проверьте соединение и настройки Firebase.»
  или похожее сообщение.

Верните правильный `projectId` и обновите страницу — сайт снова должен работать.

---

## 10. Остановка сервера

Чтобы остановить локальный сервер, в терминале, где запущен `npm run start`, нажмите:

```text
Ctrl + C
```

Ожидаемый результат: http-server завершает работу, сайт по адресу `http://localhost:5173` перестаёт открываться.
